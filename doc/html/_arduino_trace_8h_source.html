5 // A simple tracing macro to debug you program.
6 //
7 // Recipe to find where the code crashes:
8 // 1. sprinkle your code with TRACE()
9 // 2. run the program
10 // 3. view all traces in the Serial monitor
11 //
12 // Each trace includes the:
13 // * the filename
14 // * the line number
15 // * the current function
16 // * the template parameters (if any)
